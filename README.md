# DEGA DUDU-BLDC Reproducibility Repository

Reproduction code for the manuscript:

> **DEGA: A Deterministic Diagnostic Evidence Governance Agent for Industrial IoT—A DUDU-BLDC Case Study**
>
> July 15, 2026 manuscript version

The manuscript studies a deterministic **Diagnostic Evidence Governance Agent
(DEGA)** that keeps diagnostic evidence generation separate from workflow
governance. The case study uses eight DUDU-BLDC acquisitions, 200
non-overlapping 0.8-s windows, 28 current/speed features, three classifiers,
four temporal spline representations, immutable `EvidenceBundle` objects, an
external higher-priority `SafetyGuard`, hash-linked audit chains, and
deterministic replay.

This public repository is intentionally **source-only**: it contains code,
configuration, small frozen reference inputs, tests, and empty data/output
structure. It does not contain the DUDU-BLDC archive, trained models, Parquet
results, EvidenceBundles, or generated figures.

## Quick start

Requirements: Python 3.11 or 3.12 and [`uv`](https://docs.astral.sh/uv/).
Run commands from the repository root.

```bash
uv sync --all-groups
uv run pytest -q
```

Fetch the pinned public dataset:

```bash
uv run python scripts/00_fetch_dataset.py
```

Run the complete paper workflow:

```bash
uv run python scripts/reproduce_all.py
```

Generate only the manuscript result figures after experiment outputs already
exist:

```bash
uv run python scripts/06_generate_figures.py
```

The manuscript-aligned computational figures are written to
`figures_m3/output/`. They correspond to **Figures 6–14** in the attached
July 15, 2026 manuscript version.

## Stage-by-stage reproduction

| Stage | Command | Main result |
|---|---|---|
| 0 | `uv run python scripts/00_fetch_dataset.py` | Download, verify and extract pinned DUDU-BLDC data |
| 1 | `uv run python scripts/01_prepare_windows.py` | 200 canonical windows, 28-feature corpus, 16 acquisition-disjoint assignment views |
| 2 | `uv run python scripts/02_train_classifiers.py` | 48 fitted/reload-verified models, 4,800 held-out window predictions, 192 acquisition predictions |
| 3 | `uv run python scripts/03_generate_evidence.py` | 16 deviation scorers, 16 training-only Healthy references, 256 trend records, 768 EvidenceBundles |
| 4 | `uv run python scripts/04_run_xai.py` | Permutation importance, exact local LR contributions, cross-assignment stability |
| 5 | `uv run python scripts/05_run_dega.py` | 12 representative DEGA runs, SafetyGuard evaluation, audit chains and replay |
| 6 | `uv run python scripts/06_generate_figures.py` | ArticleV1 publication assets plus manuscript Figures 6–14 |

Partial execution is supported:

```bash
uv run python scripts/reproduce_all.py --from-stage 2 --to-stage 6 --skip-fetch
```

## Figure generation

The final paper version supplied with this task uses nine computational result
figures (Figures 6–14). They are generated by the dedicated publication
pipeline under `figures_m3/`:

```bash
uv run python figures_m3/scripts/make_all_figures.py
uv run python figures_m3/scripts/make_captions.py
uv run python figures_m3/scripts/validate_m3_figures.py
```

The figure scripts consume persisted experiment outputs only; they do not
retrain models or change scientific parameters. Exact plotted values are
exported under `figures_m3/output/figure_data/`.

## Important routing-policy reproducibility note

The source ZIP supplied for this repository contained the completed ArticleV1
E2 audit records but did **not** contain the historical
`Output/Models/TransitionPolicies/BayesianMCMC/posterior_trace.nc` artifact.
To keep the 12 manuscript DEGA runs reproducible, this repository includes a
small frozen routing snapshot reconstructed from those persisted E2 audit
records:

`reference_inputs/transition_policies/article_v1_routing_snapshot.json`

The default Stage 5 runner uses the original MCMC posterior if it is present;
otherwise it uses this compact snapshot to reproduce the paper's representative
workflow routing. The fallback **does not re-fit PyMC/NUTS** and must not be
reported as a new MCMC inference run. See
[`reference_inputs/README.md`](reference_inputs/README.md).

## Scientific boundaries

- The eight acquisitions are the independent source units. Windows are nested,
  correlated observations.
- The 16 exhaustive assignment views reuse the same eight acquisitions and are
  not independent datasets.
- Model probabilities are native and uncalibrated.
- The training-distribution deviation score is an OOD surrogate, not a
  validated open-world probability.
- Temporal spline derivatives are defined over normalized ordered-window
  position, not physical degradation time.
- Maximum curvature is not a validated physical failure-onset estimator.
- The study does not establish RUL, production readiness, industrial safety
  certification, or external generalization beyond the eight source
  acquisitions.
